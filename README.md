# TaThip 🚗 — ค้นหา & ติดตามยานพาหนะข้ามกล้อง CCTV

ระบบช่วยเจ้าหน้าที่ **ค้นหารถต้องสงสัยและไล่เส้นทางการเดินทางข้ามกล้องวงจรปิด** ของเมือง —
เลือกรถเป้าหมาย 1 คัน ระบบ AI คัดรถที่หน้าตาเหมือนจากทุกกล้องมาให้ **เจ้าหน้าที่ติ๊กยืนยันเอง**
(human-in-the-loop) แล้วลากเส้นทางบนแผนที่จริง

> Smart City · ขอนแก่น · prototype

---

> Backend จริงครบ: **SQLite + OCR + ประมวลผลวิดีโอตอนเพิ่มกล้อง (background + progress)** — ดู `DOCS.html` สำหรับรายละเอียดเชิงลึก

## ✨ ฟีเจอร์
- **เพิ่มกล้อง (+ ประมวลผลในตัว)** — เลือกไฟล์วิดีโอ → ปักหมุด → **OCR อ่านเวลาจริง** → วาด ROI → กดเซฟ
  → ระบบ **รัน YOLOv8 + ByteTrack + Re-ID บนวิดีโอกล้องนั้นทันที** → ได้ track เข้าระบบ (มี progress)
- **ค้นหา & ติดตาม** — กรอง (ประเภท/สี/กล้อง/วันที่/ช่วงเวลา) → เลือกรถเป้าหมาย → จัดอันดับด้วย **Re-ID cosine** → ติ๊กยืนยัน
- **Visualize** — หมุดกล้อง + รูป + เวลา · ลูกศรวิ่งตามเวลา · โหมด **Pathway**
- **บันทึกเส้นทาง** — เก็บลง SQLite เปิดดูภายหลัง
- **ตั้งค่า / ลบกล้อง** — แก้ชื่อ/พิกัด/เวลา หรือลบ (sync DB)
- **ข้อมูลถาวร** — กล้อง/เส้นทาง เก็บใน `tathip.db` (สร้างอัตโนมัติ)

## 🚀 วิธีรัน

### แบบที่ 1 — Desktop App (แนะนำ, เลือกไฟล์วิดีโอจริงได้)
```bash
pip install -r requirements.txt
python app.py
```
> Windows ใช้ Edge WebView2 (มีมากับ Win10/11) · ต้องต่อเน็ตเพื่อโหลดแผนที่

### แบบที่ 2 — เปิดในเบราว์เซอร์ (ดู UI เฉยๆ)
ดับเบิลคลิก `index.html` (ขั้นเลือกไฟล์/เฟรมจะเป็นตัวอย่าง mock)

### แพ็กเป็น .exe
```bash
pip install pyinstaller
python build_exe.py     # ได้ dist/TaThip.exe
```

## 🧠 เทคโนโลยีเบื้องหลัง
| ส่วน | ใช้ |
|------|-----|
| ตรวจจับรถ | YOLOv8 |
| ติดตามในกล้อง | ByteTrack |
| จับคู่ข้ามกล้อง | Re-ID (ResNet50-IBN / VeRi) — cosine similarity |
| อ่านเวลา | OCR (timestamp ในคลิป) |
| UI | pywebview + HTML/CSS/JS + Leaflet |

ฝั่ง AI (detect/track/Re-ID) รันเป็น pipeline แยก → ผลิตไฟล์ `tracks_data.js` ที่แอปใช้
แอปนี้คือ **ชั้นค้นหา/แสดงผล** (เริ่มต้นมาเป็นสถานะว่าง พร้อมให้เพิ่มกล้อง)

## 📌 หมายเหตุ
- แอปเริ่มมาเป็น **สถานะว่าง** — เพิ่มกล้องที่หน้า "จัดการกล้อง" แล้วระบบประมวลผลให้เอง
- **dependency การประมวลผล:** ใช้ Re-ID weights `veri_sbs_R50-ibn.pth` + `yolov8s.pt` จากโฟลเดอร์
  `BDI_hackathon/.../pipeline` (ตั้ง path ใน `app.py` ตัวแปร `BDI`) · ต้องมี GPU ถึงจะเร็ว
- ประมวลผล **5 นาทีแรกของคลิป** ต่อกล้อง (ปรับที่ `PROC_MINUTES` ใน app.py)
- ใช้ YOLOv8 **COCO** (car/motorcycle/bus/truck) — โมเดลเทรนเองไม่ใช้ (domain ไม่ตรงกล้องเรา)
- เป็น batch processing ยังไม่ใช่ real-time
